# DyKnow Skill

抖音视频 → 本地知识库 Markdown。自动下载视频、语音转录文字、生成 Obsidian 兼容笔记。

## 核心原则

**你是一个文本模型，没有视觉能力。绝对禁止用浏览器"观看"视频。**
视频内容只能通过「下载 → 语音转录 → 读取转录文字」这个管道来获取。

## 首次使用

当用户第一次使用 DyKnow 时，AI 应先确认笔记输出位置：

```
1. 询问用户："笔记文件保存在哪个目录？"
2. 如果用户未指定，默认使用项目根目录下的 `抖音收藏/`
3. 确认后告知用户目录结构
4. 确保输出目录存在，后续所有操作使用该路径
```

## 知识库目录结构

笔记采用两级管理，知识库内部按内容自动分类：

```
<输出目录>/
├── 未分类/            ← 原始下载，AI 尚未处理（status: indexed/transcribed）
└── <知识库名称>/      ← AI 已处理完毕，按主题自动归类
    ├── <分类1>/
    ├── <分类2>/
    └── ...
```

| 目录 | 含义 | 典型状态 |
|------|------|----------|
| `未分类/` | 原始下载，等待 AI 处理 | `indexed`（仅有元数据）、`transcribed`（已转录待总结） |
| `<知识库>/xx/` | AI 已处理并自动归类 | `done`（转录+章节+金句+AI摘要齐全） |

**AI 的职责**：阅读 `未分类/` 中的文件 → 生成摘要 → 根据内容自动归类 → 移动到知识库对应子目录 → 更新索引。

## 三种使用场景

### 场景A：用户粘贴带抖音链接的文字

适用：用户在聊天中发了一段含抖音链接的分享文案。

```
1. 执行: python -m dyknow parse "用户的原话"
2. 读取生成的 MD 文件（NOTE_PATH 输出）
3. 完整阅读转录文字，生成 AI 摘要，写入 MD
4. 更新 frontmatter: status: done
5. 根据内容自动归类，移动到知识库对应分类目录
6. 更新总索引文件
```

摘要格式：
```markdown
## 🤖 AI 摘要

### 一句话总结
（用一句话概括视频核心内容）

### 要点
- 要点1
- 要点2
- 要点3

### 金句
- "原文引用"
```

### 场景B：用户要同步收藏

适用："同步我的收藏"、"备份抖音内容"等。

```
1. python -m dyknow status         → 检查登录
2. python -m dyknow login          → 未登录则扫码
3. python -m dyknow sync           → 增量同步元数据索引
4. python -m dyknow sync --transcribe → 下载视频+转录（耗时！须提前告知用户）
5. python -m dyknow sync --source collection → 仅收藏夹精选
6. 转录完成后，逐条读取 MD，生成摘要，写入
7. 根据内容自动归类，移动到知识库对应分类目录
8. 更新总索引文件
```

**重要**：阶段2转录非常耗时（每条1-3分钟），必须提前告知用户预计时间。

### 场景C：主题浏览 / 自动研究

适用："帮我搜一下AI工具相关的抖音视频"、"浏览最新AI教程"等。

```
1. python -m dyknow browse -k "关键词" --count 30
   → 搜索关键词，抓取视频列表，生成索引笔记
2. python -m dyknow browse -k "关键词" --count 30 --transcribe
   → 同时下载+转录
3. 转录完成后，逐条读取 MD，生成摘要，写入
4. 根据内容自动归类，移动到知识库对应分类目录
5. 更新总索引文件
```

搜索可以指定任意关键词，相当于"用抖音搜索作为研究入口"。

## 可用命令速查

所有命令在项目根目录执行，使用 `.venv/Scripts/python.exe -m dyknow <command>`。

| 命令 | 用途 | 场景 |
|------|------|------|
| `login` | 打开浏览器扫码登录 | B |
| `status` | 查看登录+同步状态 | B |
| `parse "<文本>"` | 单视频解析 | A |
| `parse "<文本>" -o ./` | 指定输出目录 | A |
| `parse "<文本>" --no-transcribe` | 仅元数据，不转录 | A |
| `sync` | 增量同步索引 | B |
| `sync --transcribe` | 同步+转录 | B |
| `sync --source collection` | 收藏夹精选 | B |
| `sync --count 100` | 指定数量 | B |
| `sync --full` | 全量重新同步 | B |
| `browse -k "关键词"` | 主题浏览索引 | C |
| `browse -k "关键词" --transcribe` | 主题浏览+转录 | C |

## API 说明

抖音网页版不严格区分"点赞"和"收藏"。默认 `sync` 使用 `favorite` 源（用户全部互动内容）。
如需更精选的内容，使用 `--source collection`（仅收藏夹内容，数量较少但质量更高）。

## AI 知识库维护流程

当用户要求"处理收藏"或"整理知识库"时，AI 应：

```
1. 检查 未分类/ 中有多少待处理的文件
2. 优先处理已有转录文本的文件（status: transcribed）
3. 对每个文件：
   a. 完整阅读 ## 📝 完整转写 部分
   b. 生成 ## 🤖 AI 摘要（一句话总结 + 要点 + 金句）
   c. 填写 🎬 章节要点 和 💡 金句（如果还是占位符）
   d. 更新 frontmatter: status: done
   e. 根据内容自动归类，移动到知识库对应分类子目录
   f. 更新总索引文件（新增条目 + 更新统计数据）
4. 对仅有元数据的文件（status: indexed）：告知用户需要先运行转录
```

### AI 摘要的生成逻辑

AI 摘要不是简单的文本压缩，而是一次**理解 → 提炼 → 重组**的过程：

**Step 1: 完整理解**
通读 `## 📝 完整转写` 部分的全部转录文字。注意视频通常 1-5 分钟，转录文字约 500-3000 字。理解时要关注：
- 视频的核心论点是什么？作者想传达什么？
- 有哪些具体的方法、步骤、数据？
- 哪些话是作者的金句、妙语、总结性表达？

**Step 2: 提炼要点**（3-5 条）
从转录中提取最有价值的信息，原则：
- **具体 > 抽象**："用分仓法把回撤控制在 10% 以内" 比 "要控制风险" 好
- **方法 > 感受**：优先提取可操作的方法论
- **原始数据 > 加工结论**：保留作者提供的具体数字、案例

**Step 3: 提取金句**（1-3 条）
从转录中挑选作者的原话，标准：
- 表达精炼、有冲击力
- 概括了核心观点
- 可以独立引用
- 必须是原文引用，不修改措辞

**Step 4: 一句话总结**
用一句话概括"这个视频讲了什么 + 最核心的结论是什么"。避免空洞的套话，要包含具体信息。

**Step 5: 填写章节要点**
如果视频有明显的时间线或结构，为每个逻辑段落写 1-2 句要点。如果内容较短或没有明显结构，可以用一句话概括章节。

**Step 6: 写入格式**
```markdown
## 🤖 AI 摘要

### 一句话总结
（用一句话概括视频核心内容，包含具体信息）

### 要点
- 要点1：具体的方法/数据/结论
- 要点2
- 要点3

### 金句
- "原文引用1"
- "原文引用2"
```

## 自动分类规则

AI 根据视频内容自动归类，不预设固定分类列表。分类的目的是让知识库有结构、可浏览。

**分类逻辑：**
- **动态归纳**：先通览所有待处理视频的标题和转录，识别出主要的主题聚类。相近主题归入同一个分类目录
- **优先复用**：已有分类目录存在时，优先归入已有分类；内容不属于任何已有分类时，才新建
- **命名简洁直观**：用 2-6 个字命名，一眼能看出内容类型。如 `股票与量化`、`创业与副业`、`AI工具`
- **唯一归属**：一个文件只归入一个分类。跨多个主题的，选占比最大的核心话题
- **看话题不看人**：按视频讲什么分类，不受作者身份、粉丝量影响

**分类粒度控制：**
- 每个目录下至少 3 条以上内容才值得独立成类
- 少于 3 条的归类到相近目录，或暂时保留在 `未分类/`
- 一个分类下超过 30 条时，考虑拆分子分类

**示例分类体系（实际使用时动态生成）：**
```
01-AI工具与开发/
    01-Codex与ClaudeCode/
    03-大模型与部署/
    04-Agent与工作流/
04-商业与投资/
    01-创业与副业/
    02-股票与量化/
    03-赚钱思维与认知/
```

## 知识库总索引

处理完成后，AI 在知识库根目录维护一个总索引文件 `知识库索引.md`。

**索引的结构：**
```markdown
# 知识库索引
> 最后更新：日期 | 已精炼 X 条 | 待转录 Y 条

## 概览（分类统计表）
## 🤖 AI工具与开发（已精炼 X 条）
  ### 视频标题
  - 作者 / 点赞数
  - 一句话：...
  - 要点：...
  - 金句：...
  <details><summary>待转录（Y 条）</summary>...</details>
## 🎬 AI视频创作 ...
...
```

**索引的生成逻辑：**

1. **概览表** — 统计每个分类的 `done` 数量和 `transcribed/pending` 数量，一眼看出知识库规模
2. **已处理条目** — 从笔记的 `## 🤖 AI 摘要` 部分提取「一句话总结 + 要点 + 金句」，精简呈现
3. **待处理条目** — 折叠在 `<details>` 中，仅列出标题+作者+点赞数，等待后续转录和总结
4. **更新策略** — 每处理完一批笔记后，重新生成索引文件（不是增量编辑，而是全量重建）

**索引的设计原则：**
- **精简至上**：收藏量会持续增长，索引的目的是快速定位，不是替代完整笔记
- **金句点睛**：1-2 条金句比长篇摘要更能帮人回忆起视频内容
- **链接直达**：每个条目可链接到完整笔记文件
